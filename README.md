# belarus-legal-mcp

MCP-сервер для Claude Desktop, который добавляет инструменты для работы с белорусским законодательством:

- **pravo.by** — скачивание и поиск по PDF с автопроверкой актуальности редакции
- **ilex.by** — поиск и чтение документов через авторизованную сессию Chrome

## Инструменты

| Инструмент | Описание |
|-----------|---------|
| `crawl` | Скрапинг HTML-страницы → Markdown |
| `search_crawl` | Найти релевантные фрагменты на HTML-странице (нужно для объёмных карточек pravo.by — иначе результат может превысить лимит размера ответа) |
| `download_pdf` | Скачать PDF целиком (с кешем и проверкой редакции для pravo.by) |
| `search_pdf` | Найти релевантные фрагменты в PDF (~97% экономии контекста) |
| `search_ilex` | Поиск документов на ilex.by по запросу, включая тематические таблицы/классификаторы |
| `search_ilex_document` | Найти до 5 релевантных фрагментов в документе ilex.by; при явном номере статьи/пункта автоматически возвращается точный структурный элемент |
| `get_ilex_sections` | Пакетно получить точные статьи/пункты документа ilex.by по номерам |
| `crawl_authenticated` | Чтение страницы ilex.by через вашу сессию Chrome (headless) |
| `get_pdf_sections` | Пакетно получить точные статьи/пункты PDF по номерам |

## Установка

### macOS

```bash
# 1. Создать окружение
python3 -m venv ~/.claude/mcp_servers/crawl4ai_env
~/.claude/mcp_servers/crawl4ai_env/bin/pip install crawl4ai mcp pypdf httpx pymorphy3 playwright striprtf
~/.claude/mcp_servers/crawl4ai_env/bin/playwright install chromium

# 2. Скопировать сервер
cp server.py ~/.claude/mcp_servers/crawl4ai_server.py
```

Добавить в `~/Library/Application Support/Claude/claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "crawl4ai": {
      "command": "/Users/ИМЯ/.claude/mcp_servers/crawl4ai_env/bin/python",
      "args": ["/Users/ИМЯ/.claude/mcp_servers/crawl4ai_server.py"]
    }
  }
}
```

### Windows

```cmd
python -m venv %USERPROFILE%\.claude\mcp_servers\crawl4ai_env
%USERPROFILE%\.claude\mcp_servers\crawl4ai_env\Scripts\pip install crawl4ai mcp pypdf httpx pymorphy3 playwright striprtf
%USERPROFILE%\.claude\mcp_servers\crawl4ai_env\Scripts\playwright install chromium
```

Добавить в `%APPDATA%\Claude\claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "crawl4ai": {
      "command": "C:\\Users\\ИМЯ\\.claude\\mcp_servers\\crawl4ai_env\\Scripts\\python.exe",
      "args": ["C:\\Users\\ИМЯ\\.claude\\mcp_servers\\crawl4ai_server.py"]
    }
  }
}
```

> Замените `ИМЯ` на ваше имя пользователя.

Перезапустить Claude Desktop.

## Использование

### Работа с pravo.by

```
Скачай документ: https://pravo.by/upload/docs/op/W21226212_1344459600.pdf
```

```
Найди в этом PDF всё про персональные данные:
url: https://pravo.by/upload/docs/op/W21226212_1344459600.pdf
query: персональные данные
```

Сервер автоматически:
- Кеширует PDF на диск (`~/.claude/mcp_servers/pdf_cache/`)
- При каждом обращении проверяет, не вышла ли новая редакция на pravo.by
- Уведомляет, если кеш был обновлён

### Работа с ilex.by

Нужна активная авторизованная сессия Chrome.

```
Найди на ilex: статья 169 трудовой кодекс
```

```
Открой документ: https://ilex-private.ilex.by/view-document/BELAW/219268/
```

Для точечного вопроса по конкретному документу — без загрузки полного текста:

```
Найди в этом документе про уведомление об отпуске:
url: https://ilex-private.ilex.by/view-document/BELAW/219268/
query: уведомление о начале отпуска
```

Если номера норм уже известны, выгоднее получить их одним пакетным вызовом:

```text
url: https://ilex-private.ilex.by/view-document/BELAW/219268/
sections:
  - статья 18
  - статья 261-3
```

Тематические поисковые инструменты по умолчанию возвращают не более 5
фрагментов и соблюдают мягкий бюджет 12 000 символов. Первая найденная норма
никогда не обрезается посередине; не поместившиеся элементы перечисляются в
ответе и могут быть запрошены отдельно.

Chrome при этом визуально не открывается — всё работает headless. Текст документа кешируется на диск; при каждом обращении сервер лёгким запросом (без экспорта в Word) проверяет дату редакции в заголовке страницы и обновляет кеш, если она изменилась — так же, как для pravo.by. Принудительно обновить: `bypass_cache: true`.

## Системный промпт

Системный промпт **не заменяет** MCP-сервер — без него инструменты просто не появятся у Claude. Промпт лишь указывает Claude *когда и как* их использовать (например, автоматически идти в ilex.by при неполном тексте на pravo.by). Нужны оба компонента.

Рекомендуется добавить в системный промпт Claude. Пошаговый чек-лист работает
надёжнее, чем описание прозой — Claude иногда пропускает шаги («не удалось
получить полный текст, но вот что нашлось в вебе») вместо перехода к следующему
инструменту, если инструкция сформулирована как общее пожелание, а не строгая
последовательность:

```
При вопросах о законодательстве Республики Беларусь используй только первичные тексты pravo.by, а если применяемую норму там невозможно получить или проверить, — первичные документы BELAW на ilex.by. Выводы по памяти, поисковым сниппетам и вторичным источникам запрещены.

1. Сначала определи минимальный перечень документов, норм и условий их применимости. Если норма называет роль, статус или несколько альтернатив, отдельно проверь требования к каждой из них. Не переноси ограничения одной альтернативы на другую.

2. Для каждого документа сначала найди карточку на pravo.by. Для установления URL допускаются не более двух точных поисковых запросов. Выдачу используй только для URL или регистрационного номера. Не угадывай guid, p0 и другие идентификаторы.

3. На карточке проверь название и реквизиты, статус, дату вступления в силу, действующую редакцию, наличие актуального консолидированного текста, даты изменений и последний вступивший в силу акт, изменивший конкретную применяемую статью, пункт или часть, а не только документ в целом. Будущие изменения не применяй.

4. Если номера норм известны, используй get_pdf_sections или get_ilex_sections и запрашивай несколько норм одного документа одним вызовом. Если номера неизвестны, используй search_crawl, search_pdf или search_ilex_document. Если инструмент сообщил, что запрошенная норма не помещена в лимит, считай ее неполученной и при необходимости запроси отдельно до ответа. Для PDF передавай referer корневого домена. Полный текст загружай только когда точечного поиска недостаточно; один документ полностью загружай не более одного раза.

5. Если на pravo.by невозможно получить или проверить норму, переходи к ilex.by: search_ilex → get_ilex_sections либо search_ilex_document → при необходимости crawl_authenticated. Используй первичные документы BELAW. Если первичный текст недоступен, прекрати исследование и сообщи, какой документ или инструмент недоступен.

6. До вывода проверь, могут ли устав, договор, протокол, локальный акт, статус лица или иные неизвестные обстоятельства изменить результат. Если такой документ или факт не проверен, категоричный вывод запрещен: краткий вывод должен быть условным, прямо называть зависимость и содержать только условные варианты. Не выдавай предположения за факты.

7. Ответ допускается только после получения текста каждой применяемой нормы. Укажи документы и нормы, статус и действующую редакцию, последний изменяющий акт, прямые ссылки, установленные факты, существенные допущения и неполученные документы, способные изменить вывод. Не ссылайся на норму, текст которой не был получен.

Без отдельного разрешения запрещены: etalonline.by, consultant.ru, garant.ru, pravo.gov.ru, иные сайты госорганов, статьи, комментарии, форумы и другие вторичные источники.
```

## Как это устроено: ilex.by и виртуальный скроллинг

Большие документы на ilex.by рендерятся с виртуальным скроллом — в DOM браузера
всегда присутствует только видимая на экране часть, остальное подгружается и
выгружается по мере прокрутки. Из-за этого прямое чтение текста со страницы
(`page.inner_text()`) обрезает документ до нескольких первых экранов — для
документа на 1200+ пунктов можно получить лишь ~100 из них.

Решение: сервер кликает по кнопке «Экспорт в Word» на странице документа,
перехватывает скачивание получившегося `.rtf`-файла и конвертирует его в текст.
Это даёт полный документ вместо обрезанного фрагмента. На macOS конвертация
идёт через встроенный `textutil`; на других платформах — через библиотеку
`striprtf` (даёт менее точный результат на документах с крупными таблицами).

## Требования

- Python 3.10+
- Google Chrome (установленный, для `crawl_authenticated`, `search_ilex` и `search_ilex_document`)
- Подписка на ilex.by с активной сессией в Chrome (для ilex-инструментов)
- macOS — для наиболее точного извлечения текста документов ilex.by (`textutil`);
  на других ОС используется библиотека `striprtf` (`pip install striprtf`)
